Le variabili

Servono per memorizzare dei valori

Ogni variabili è identificata da un nome che segue queste regole:

  • Niente spazi, niente trattini -, niente caratteri speciali (anche ? oppure .)
  • deve iniziare con una lettera, sono amessi anche undercore_ e il dollaro $
  • Sono case sensitive
  • Non può inizare con un numero
  • non deve mai coincidere con le keyword di Js (i, else, while, switch ecc..)

Dichiarare impcicità delle variabili

Non è obblicatorio dichiarare le variabli, cioè fare un'operazione espicità di creazione

Quando c'èe l'identificazione = l'engine crea implicitamente la variavile se non esiste già

Var, let const

Var è la prima keyword creata da javascript per definire una variabile, var è accessibile in qualeunque parte del codice script

Let è stata indtodotto da ECMaScript 6, e accessibile solo in in un determinato ambito (scope) delimitato da un blocco di codice

Un blocco di codice js è una serire di istruzione delimitata da (...blocco di codice...)

const è stata indtodotto da ECMaScript 6, -> istruisce l'engine e il valore assegnato è modificabile

Strict mode

E' una operazione di controllo del codice, è stata indtodotto da ECMaScript 5, -> se viene dichiarato lo strict mode javascript viene eseguito usando regole più stringenti-> ad esempio se una variabile non viene dichiarata ci viene mostrato un errore -> se dichiriamo lo strict mode -> regole sono meno permessive -> stile di programmazione
